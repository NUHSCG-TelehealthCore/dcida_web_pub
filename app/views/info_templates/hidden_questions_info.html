Hidden questions, called data sources can be used to individualize your tool for each user. You can use them to hook up your tool to remote databases (e.g. REDCap or MySQL), inject dynamic data based on previous question responses (e.g. lookup tables), or perform calculations.

There are a variety of different question response types that can be "hidden". When a question is hidden, the value for that question's response each time a user accesses your tool must come from elsewhere. Depending on the question response type, this can be from one or more places. The way in which this value is determined for each question response type is outlined below.

Using calculations

All radio questions have a Numeric Value that can be associated with a response. Numeric values from multiple questions can be combined using the Question Response Value Calculation field that appears when you activate the Hide Question checkbox for a radio, yes/no, or numeric question.

Example 1: Three-item stroke scale

Assume that you are developing a risk calculator to determine an incoming patient's stroke severity. You want to use the 3-item stroke scale (original paper).

You would start by defining the three questions corresponding to the 3 items. See the table below. Note the "Question ID" for each question as those are important when creating the calculations.

Disturbance of consciousness in the patient? (Question ID = 1)

Response value Numeric value
Absent 0
Moderate 1
Severe 2

With these three questions defined, you could now create a fourth radio question, but tick the Hide Question checkbox. Ticking that checkbox will cause a new field to appear, called the Question Response Value Calculation. You are now ready to input an equation here.

Using the button, you could select each question, and join them using the operator. Given the IDs above, your calculation would be displayed like this:

[question_1] + [question_2] + [question_3]

The Question Content field could be something like: "3-item stroke score".

Finally, you would need to add responses for each possible outcome of the 3-item stroke scale. Since the total score is a number from 0-6, you would have 7 different responses defined here. The key is that the calculations use the Numeric Value of each question response, and also look up the resulting response based on the Numeric Value field of the hidden question. The responses of the hidden question could look like the table below:

Question
3-item stroke score

Stroke score of 0 0
Stroke score of 1 1
Stroke score of 2 2
Stroke score of 3 3
Stroke score of 4 4
Stroke score of 5 5
Stroke score of 6 6

You now have a hidden question that gets it's response value from the responses to other questions. You could export this value using the data export features, or inject it at at a later point in your decision aid. You could also filter options based on the result of this hidden question. You could even use this result as an input into other hidden questions. Hidden questions can be extremely powerful - it just takes some creativity to imagine how they can connect various pieces of your tool together!
